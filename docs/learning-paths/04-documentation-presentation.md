## 第二章：如何撰写技术报告

### 2.1 技术报告结构

| 章节 | 内容 | 篇幅 |
|------|------|------|
| 摘要 | 研究目的、方法、主要结论 | 200字 |
| 引言 | 研究背景、问题、贡献 | 500字 |
| 相关工作 | 前人方法、我们的改进 | 800字 |
| 方法 | 核心算法、模型设计 | 1500字 |
| 实验 | 数据集、设置、结果 | 1200字 |
| 结论 | 总结、局限、未来工作 | 300字 |

### 2.2 摘要撰写技巧

**摘要公式**：
```
本文[研究背景]，
提出[方法名称]，
通过[核心技术]，
在[数据集]上达到[性能指标]。
```

**示例**：
```
本文针对序列推荐任务，提出基于Transformer的自注意力模型SASRec，
并引入时间感知机制TiSASRec和流形约束超连接mHC。
在MovieLens 1M数据集上，NDCG@10达到0.4512，HR@10达到0.7689，
相比基准模型提升8%和5%。
```

### 2.3 方法章节要点

**结构**：
1. 问题定义（定义要解决什么问题）
2. 模型框架（整体架构图）
3. 核心模块（每个模块详细解释）
4. 训练细节（损失函数、优化器等）

**写作技巧**：
- 用图代替文字描述
- 用公式表达核心思想
- 用代码片段辅助说明

### 2.4 实验章节要点

**结构**：
1. 数据集介绍
2. 评估指标
3. 基准模型
4. 消融实验
5. 结果分析

**结果展示**：

```
表1：主要结果对比
┌──────────────┬──────────┬──────────┐
│ 模型         │ NDCG@10  │ HR@10    │
├──────────────┼──────────┼──────────┤
│ SASRec       │ 0.4123   │ 0.7234   │
│ + TiSASRec   │ 0.4289   │ 0.7456   │
│ + mHC        │ 0.4356   │ 0.7512   │
│ 完整模型     │ 0.4512   │ 0.7689   │
└──────────────┴──────────┴──────────┘

结论：时间信息提升X%，mHC提升Y%
```

## 第三章：PPT制作指南

### 3.1 推荐PPT结构（10-15页）

| 页数 | 内容 | 时间 |
|------|------|------|
| 1 | 封面 | 1分钟 |
| 2-3 | 项目背景与问题 | 2分钟 |
| 4-5 | 方法概述 | 3分钟 |
| 6-8 | 核心模块详解 | 4分钟 |
| 9-10 | 实验结果 | 3分钟 |
| 11 | 消融实验 | 2分钟 |
| 12 | 总结与展望 | 1分钟 |

### 3.2 每页内容要点

**封面页**：
- 项目名称
- 团队成员
- 日期

**问题页**：
- 用一个例子说明问题
- "传统方法 vs 我们的方法"

**方法页**：
- 画架构图
- 标注核心创新点

**结果页**：
- 用表格/图表
- 突出改进效果

### 3.3 PPT设计原则

| 原则 | 说明 | 示例 |
|------|------|------|
| 少文字 | 每页<30字 | "Transformer架构" |
| 多图 | 一图胜千言 | 架构图、流程图 |
| 突出重点 | 关键数字标红 | "NDCG@10: **0.4512**" |
| 统一风格 | 字体、颜色一致 | 蓝色主题 |

### 3.4 演讲技巧

**时间控制**：
- 10页PPT → 约10分钟
- 每页平均1分钟
- 留2分钟问答

**语言表达**：
- 开头："今天我要介绍..."
- 过渡："接下来..."
- 强调："需要特别注意的是..."

## 第四章：常见答辩问题及回答

### 4.1 模型相关问题

**Q1: 为什么选择Transformer而不是RNN？**

**参考回答**：
"Transformer相比RNN有两个优势：第一，可以并行计算，速度快；第二，自注意力机制可以直接建模任意位置之间的依赖，不会像RNN那样丢失长距离信息。在序列推荐任务中，用户行为可能跨越很长时间，Transformer能更好地捕捉这种长期依赖。"

---

**Q2: mHC相比标准残差连接有什么优势？**

**参考回答**：
"标准残差连接是简单的相加：x + F(x)。在深层网络中，这可能导致信号不稳定。mHC通过Sinkhorn-Knopp算法将残差映射约束到双随机矩阵流形，确保输出是输入的凸组合，保持信号范数。这使得模型在更深的网络下也能稳定训练，并且通过多流信息交换提升性能。"

---

**Q3: TiSASRec的时间信息是如何融入的？**

**参考回答**：
"传统注意力只考虑内容相似度：Q·K^T。TiSASRec在注意力计算中加入时间间隔项：Q·T·Q，其中T是时间间隔矩阵。这使得模型不仅学习'用户买了什么'，还学习'什么时候买的'，越近的行为影响越大。"

### 4.2 实验相关问题

**Q4: 为什么选择NDCG和HR作为评估指标？**

**参考回答**：
"HR@10衡量的是'推荐列表中是否包含目标物品'，直接反映用户体验。NDCG@10还考虑排名位置，用户更希望看到目标物品排在前面。两个指标结合能全面反映推荐质量。"

---

**Q5: 消融实验说明了什么？**

**参考回答**：
"消融实验显示：单独加时间信息提升约4%，单独加mHC提升约5%，两者结合提升约9%。这说明时间信息和mHC是正交的，可以互补。"

---

**Q6: 模型有什么局限性？**

**参考回答**：
"当前模型有几个局限：第一，计算复杂度O(n²)，对超长序列有挑战；第二，没有考虑物品的内容信息（如文本、图像）；第三，冷启动问题仍然存在。"

### 4.3 扩展问题

**Q7: 如果有更多时间，会做什么改进？**

**参考回答**：
"如果有更多时间，我想尝试：第一，结合物品内容的预训练表示；第二，探索更高效的时间编码方式；第三，在更大规模数据集上验证。"

---

**Q8: 这个项目可以应用到哪里？**

**参考回答**：
"这个项目的技术可以应用到任何需要预测用户下一步行为的场景：电商推荐、视频推荐、音乐推荐、新闻推荐等。TiSASRec的时间感知机制特别适合用户行为间隔差异大的场景。"

## 第七章：答辩注意事项

### 7.1 答辩前准备

| 检查项 | 状态 |
|--------|------|
| PPT在目标电脑能打开 | ☐ |
| 演示环境已测试 | ☐ |
| 备用方案（纸质材料） | ☐ |
| 常见问题答案准备好 | ☐ |
| 睡眠充足 | ☐ |

### 7.2 答辩现场技巧

**开场**：
- 微笑，自信
- "各位老师好，我是..."

**演示**：
- 语速适中，不要太快
- 适时停顿，让听众消化
- 指着屏幕说"这里"

**问答**：
- 听清问题再回答
- 没听清可以让老师再说一遍
- 不会的问题诚实的说"这个我不太了解"

### 7.3 常见错误避免

| 错误 | 正确做法 |
|------|---------|
| PPT文字太多 | 每页<30字 |
| 照念PPT | 用自己的话讲解 |
| 过度谦虚 | 客观陈述事实 |
| 过度自信 | 承认局限性 |
| 时间不够 | 多次练习，控制时间 |
